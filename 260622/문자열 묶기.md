## 💡 문제 해결 전략

뭔가 복잡해보여서 풀 수 있을까 했는데

일단 구현해보고 나중에 간단하게 리팩토링을 시도하면 된다는 생각으로 풀기 시작했다.



1)

for문이 해결 과정에 쓰일 것 같다는 생각이 들었다.

배열의 길이를 100,000으로 하자니 for문 반복을 많이 할 것 같았다.

마침 strArr의 각 문자열의 길이가 최대 30이므로, 크기가 30인 배열을 선언하기로 하였다.

int [] strCount = new int[30];



2)

"길이가 s.length()인 문자열의 개수 증가하면,

strCount[s.length()] 값도 1 증가할 것이다."를 for문으로 작성했다.



3)
이제 strCount의 원소 중에서 가장 큰 원소를 구하면 되는데

뭔가 배열의 최댓값을 구하는 메서드가 있을 것 같긴하지만,

현재 그 메서드를 모르기 때문에 for문으로 최댓값을 구하기로 하였다.



4)

다음과 같이 작성하였으나, 런타임 에러가 발생하였다.

```java
class Solution {
    public int solution(String[] strArr) {
        // (길이로 묶었을 때) 각 문자열의 개수 배열 strCount 구하기
        int [] strCount = new int[30];
        for(String s : strArr) {
            // 길이가 s.length()인 문자열의 개수 증가
            strCount[s.length()]++;
        }

        // strCount의 정수 중에서 최댓값 구하기
        int answer = strCount[0];
        for (int k : strCount) {
            if (k > answer) {
                answer = k;
            }
        }

        // 최댓값 반환
        return answer;
    }
}
```

물론 프로그래밍 문제 사이트에서는 "런타임 에러"라고만 나오기는 했지만,

배열을 다룰 때마다 자주보는

index out of bounds exception 런타임 에러가 떴을 확률이 높아 보였다.



문자열 길이의 최대값이 30인데

배열의 마지막 인덱스는 29라서 `strCount[30]` 접근 시 예외가 발생했던 것으로 보인다.



new int[30]으로 설정한 배열의 인덱스는 0부터 시작하여 30-1까지라는 것을 알지 못해 생긴 실수였다.

이를 고려하여 소스를 바꾸었고, 그 결과 통과했다.

## 💻 코드

```java
class Solution {
    public int solution(String[] strArr) {
        // (길이로 묶었을 때) 각 문자열의 개수 배열 strCount 구하기
        int [] strCount = new int[30];
        for(String s : strArr) {
            // 길이가 s.length()인 문자열의 개수 증가
            strCount[s.length() - 1]++;
        }

        // strCount의 정수 중에서 최댓값 구하기
        int answer = strCount[0];
        for (int k : strCount) {
            if (k > answer) {
                answer = k;
            }
        }

        // 최댓값 반환
        return answer;
    }
}
```

다른 사람의 풀이는 다음과 같다.  

```java
class Solution {
    public int solution(String[] strArr) {
        int[] num = new int[30+1];
        for(String s : strArr){
            num[s.length()]++;
        }

        int answer = 0;
        for(int n : num){
            answer = Math.max(answer, n);
        }
        return answer;
    }
}
```

위 풀이처럼 문자열의 길이를 그냥 처음부터 30+1로 하는게 더 좋아 보였다.



strCount[1] = 길이 1 문자열 개수
strCount[2] = 길이 2 문자열 개수
...
strCount[30] = 길이 30 문자열 개수



이렇게 설명하는게 더 직관적이고 설명하기 쉽기 때문이다.



 나의 풀이 처럼 strCount[s.length() - 1]++; 로 하는 건

다른 사람이 보기에도 내가 보기에도 계속 헷갈린다. 



추가로 Math.max 메서드를 활용하면 최댓값을 구할 수 있다는 것을 알게 되었다.
