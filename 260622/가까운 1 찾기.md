## 💡 문제 해결 전략

1)

먼저 for문으로 해당 조건을 만족하면 바로 early return을 하고,

for문으로 arr의 모든 원소에 대한 검사가 끝난 상태에서는 -1을 반환하면 되겠다는 생각이 들었다.



2)

idx가 arr의 길이보다 작은 경우, 해당 조건을 만족하는 인덱스는 없으므로

메서드의 초반부에서 이를 검사하여 -1을 반환하기로 했다.



3)

idx보다 크거나 같은 인덱스를 찾는 것이므로,

for문에서 인덱스는 idx보다 시작하도록 정했다.

그리고 arr의 원소 끝까지 검사를 해야 하므로 k<arr.length를 for문의 조건부에 두었다.

## 💻 코드

```java
class Solution {
    public int solution(int[] arr, int idx) {
        // idx가 arr의 길이보다 작은 경우, 해당 조건을 만족하는 인덱스는 없음
        if (idx>arr.length) {
            return -1;
        }

        //idx부터 arr.length까지 조건을 만족하는 인덱스가 있는지 검사
        for(int k=idx; k<arr.length; k++) {
            if (arr[k]==1) {
                return k;
            }
        }

        // 해당 조건을 만족하는 인덱스가 없음
        return -1;
    }
}
```

```java
class Solution {
    public int solution(int[] arr, int idx) {
        for (; idx < arr.length; idx++) {
            if (arr[idx] == 1)
                return idx;
        }
        return -1;
    }
}
```

항상 for문의 정해진 형식을 꼭 따라야 한다고 생각했는데, 아니었다.

for문의 변수 초기화 부분은 굳이 없어도 된다는 것을 알게 되었다.


다른 사람의 풀이를 보고 나니
