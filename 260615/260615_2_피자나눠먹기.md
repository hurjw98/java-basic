## 💡 문제 해결 전략

매개변수의 범위가 제한되어 있어 추가하였습니다.

그리고 처음에는 n%7==0인 것을 생각하지 않고 n/part를 반환했는데, 그렇게 하니 3번 케이스에서 에러가 발생하였고

이 에러를 통해 소스를 고칠 수 있었습니다.

이 문제를 통해 테스트 케이스가 중요함을 다시 알았습니다.

## 💻 코드

```java
class Solution {
    public int solution(int n) {
        int part = 7;
        int answer = 0;
        if(n>0 && n<=100) {
            if (n <=7) {
                return 1;
            } else if (n%7==0) {
                return n/part ;
            } else {
                return (int) n/part + 1;
            }
        }
        return answer;
    }
}
```
